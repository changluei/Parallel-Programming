\section*{摘要}
本文面向这一开放式题目要求，对我国超级计算机发展历史做系统调研，并突出并行体系结构的机制分析。本文讲解了从银河早期巨型机、曙光与深腾时代的集群化探索，到天河-1A/天河-2/天河-2A的异构工程化，再到神威太湖之光与近年公开在榜系统的持续演进。随后，本文选取神威太湖之光与天河-2A作为代表进行剖析，重点讨论节点内并行组织、数据搬移机制、节点间互连与软件协同。最后基于 2025 年 11 月 TOP500/HPCG/Green500 的公开信息，与 El Capitan、Frontier、Aurora 进行对比，指出我国未来突破关键在于“架构—内存—互连—软件生态”全栈协同，而非单一峰值指标竞争。

\noindent\textbf{关键词：}超级计算机；历史发展；并行体系结构；发展方向；Exascale

\section{引言：调研目标与分析框架}
高性能计算进入 Exascale 时代后，系统竞争已从“单机峰值”转向“系统协同效率”。仅用 HPL 峰值评价系统强弱会忽略三个关键层面：  
\textbf{节点计算组织}（核/加速器分工）、\textbf{数据供给效率}（内存层次与搬移开销）、\textbf{系统通信能力}（互连拓扑与软件栈协同）\cite{top500-2025-11,top500-hpcg-2025-11,green500-2025-11}。

这条时间线真正值得强调的，不是“每隔几年出现一个新型号”，而是系统目标的持续升级。银河时期首先解决的是“有没有”的问题，即能否在核心器件、系统集成和工程组织都较为薄弱的条件下完成可运行、可验证的高性能系统；进入曙光与深腾等集群化探索阶段后，重点转向“能不能把规模做起来且稳定运行”，这本质上是并行系统工程能力的建设期。到天河-1A、星云进入国际头部榜单时，我国超算开始从“国内技术突破”转向“国际同台竞争”，评价口径也由单机能力扩展到整机可交付能力与长期运行能力。再到天河-2/2A与神威太湖之光阶段，问题不再是单次冲榜，而是如何形成可持续迭代的体系结构路线：一条强调异构协同与工程可迁移性，一条强调众核组织与架构自主控制力。近年公开在榜系统延续的正是这种从“点状突破”走向“体系化演进”的轨迹。

\subsection{历史演进规律：每一代系统解决的“主问题”不同}
从机制上看，我国超算历史不是“代际替换”，而是“主问题迁移”。
早期阶段的核心矛盾是可用性：系统能否独立研制、稳定运行、具备工程复现能力。
这个阶段的技术价值不在绝对峰值，而在把设计、制造、系统软件、运维组织等环节串成闭环。
第二阶段的主问题转为规模并行与工程交付，即系统不仅要跑得快，还要在更大规模下保持稳定效率，并在公开评价体系中证明竞争力。天河-1A、星云、天河-2系列的意义，正在于把并行计算从“实验室突破”推进到“国家级工程能力展示”。第三阶段的主问题进一步转向可持续演进：当外部技术环境和应用需求同时快速变化时，
超算竞争已不再是一次性性能冲刺，而是体系结构、互连网络、编译运行时与应用生态的联动迭代能力。

如果把这一演化链放在并行体系结构视角下观察，还能看到一个更深层的规律：每一代系统都在重新划分“硬件负责什么、软件负责什么”。早期系统更多依赖硬件层面的能力堆叠；进入大规模并行阶段后，通信库、任务划分和负载均衡的重要性迅速上升；到近年体系化发展阶段，性能上限越来越取决于全栈协同，而不是某一个芯片参数。这个规律解释了为什么同样的峰值算力在不同系统上的应用表现会出现显著差异，也解释了为什么当前评估超算发展不能只看榜单名次，而必须结合应用效率、能效和可持续演进能力进行综合判断。

\subsection{为什么会形成多路线并进}
2010 年后形成“天河系异构工程路线”与“神威系自主众核路线”并进，并不是重复建设，而是对不同约束条件的理性回应。异构工程路线强调在既有高性能应用生态中实现较低迁移成本和稳定交付，它通过 CPU 与加速器分工、专有互连优化和系统级调度协同，在较大程度上兼顾了性能、可用性与工程风险控制；这一逻辑适合应用类型复杂、交付周期明确、需要持续保障服务能力的场景。自主众核路线则把重点放在架构控制力和并行密度潜力，通过管理核与计算核阵列的组织方式追求高吞吐与高能效，同时把更多优化空间交给软件侧深度调优；这种路线在规则负载和定制优化充分时具备突出优势，但对编程模型、开发团队能力和长期生态投入提出更高要求。

因此，多路线并进的本质不是“谁替代谁”，而是“不同路线覆盖不同问题空间”。从国家超算体系建设角度看，单一路线往往会在某一类约束下出现结构性短板，而并行布局能够分散技术风险、扩大应用覆盖面，并为后续技术融合保留空间。更重要的是，两条路线在实践中并非封闭发展：异构路线持续吸收自主优化能力，自主众核路线也在不断强化工程工具链与应用迁移方法。这个相互借鉴的过程，本身就是我国超算从“追赶式建设”走向“体系化创新”的关键标志。

\section{代表系统并行体系结构深析}
\subsection{神威太湖之光：众核分组架构的效率来源与代价}
神威太湖之光采用 SW26010 众核处理器，其核心思想不是强化单核复杂度，而是通过“管理核 + 计算核阵列”实现高密度并行\cite{top500-sunway,top500-2016-highlights}。  
其效率来源主要有三点：
\begin{enumerate}[leftmargin=2.5em]
  \item \textbf{分工清晰：} 管理核承担控制与通信，计算核阵列承担高吞吐数值计算；
  \item \textbf{数据流可控：} 强调显式数据搬移，降低通用缓存一致性开销；
  \item \textbf{层次并行：} 组内细粒度并行 + 组间/节点间消息并行。
\end{enumerate}
但其代价同样清晰：编程门槛较高，应用迁移与调优更依赖经验型优化团队。换言之，该路线把一部分硬件复杂度转移到了软件优化复杂度。

\subsection{天河-2A：异构工程路线的系统平衡能力}
天河-2A在公开资料中体现出“CPU + 加速器 + 专有互连”协同模式\cite{top500-tianhe2a,liao2015-th-express}。  
其优势不在某个部件绝对领先，而在系统平衡：
\begin{enumerate}[leftmargin=2.5em]
  \item \textbf{任务类型匹配：} CPU 负责复杂控制，加速器负责高并行算子；
  \item \textbf{通信组织优化：} 互连设计降低大规模同步与远程通信开销；
  \item \textbf{工程可迁移性：} 与主流异构开发模型相近，应用改造成本相对可控。
\end{enumerate}
这条路线体现了“工程协同驱动”的方法论：即使单点参数不最激进，也可以依靠系统集成能力获得稳定的持续性能。

\section{进阶对比：与国际最新 Exascale 系统比较}
\subsection{国际头部系统的共同结构特征}
通过对公开榜单进行调研，El Capitan、Frontier、Aurora 位列公开榜单前列\cite{top500-2025-11,llnl-elcapitan,olcf-frontier,alcf-aurora}。其共同特征是：
\begin{enumerate}[leftmargin=2.5em]
  \item 深度异构（CPU 与 GPU/APU 紧耦合）；
  \item 高带宽内存主导的数据供给体系；
  \item 互连硬件、通信库、调度策略协同设计；
  \item 成熟的软件生态支撑大规模应用迁移。
\end{enumerate}

\subsection{差距的结构来源：不是“单参数落后”}
若只看 HPL，差距显著；但更关键的是差距来源：
\begin{enumerate}[leftmargin=2.5em]
  \item \textbf{封装与内存：} 高带宽内存和先进封装直接决定“算力兑现率”；
  \item \textbf{互连与系统软件：} 在万节点级规模下，通信效率常先于计算峰值成为瓶颈；
  \item \textbf{开发生态：} 编译器、运行时、性能分析工具决定应用落地速度。
\end{enumerate}
因此，国际竞争焦点已从“谁峰值更高”转向“谁全栈协同更成熟”。

\section{面向下一阶段的建议}
\subsection{建议一：以关键应用反推体系结构}
从“先造硬件再找应用”转向“以应用牵引架构”，围绕气象、材料、生命科学、AI for Science 等重点领域做联合优化。

\subsection{建议二：把数据流效率作为一等指标}
未来提升应重点关注“有效带宽利用率、通信重叠率、实际并行效率”，而不仅是峰值 FLOPS 增长。

\subsection{建议三：硬件迭代与生态建设并重}
同步强化编译器、通信库、性能分析和自动调优工具，把“专家可优化”转化为“团队可复现”。

\section{结论}
本文按“扩展时间线 + 机制分析 + 国际对比”的方式，系统回答了我国超算发展历史问题。核心结论是：  
我国超算发展不是“银河、天河、神威、曙光”四点并列，而是一条持续演进的历史链；在此过程中形成了以天河为代表的异构工程路线与以神威为代表的自主众核路线。面向未来，决定竞争力上限的关键将是架构、内存、互连与软件生态的全栈协同能力。
